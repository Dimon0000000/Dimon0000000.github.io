线段树：

线段树是什么？

线段树是一种基于分治思想的二叉树结构，用于再区间上进行信息统计。

  1. 线段树的每一个节点都代表一个区间
  2. 线段树具有唯一的根节点，代表的区间为**[i, N]**
  3. 线段树内的每个叶节点都代表一个长度为1的元区间**[x, x]**
  4. 对于每一个内部节点**[l, r]，它的左子节点为[l, mid]，右子节点为[mid + 1, r]**

线段树用来做什么？

线段树常常用于处理「区间和」的问题，且该「区间」会发生变化

在上图的线段树中，除去最后一层之后，整颗线段树为一完全二叉树，二叉树的深度为O(longN)

下面我们来逐个分析线段树中各个方法的作用这里以「区间更新」的线段树为例

这部分为线段树的节点：

  • ls, rs分别为某一节点的左，右节点
  • val为当前线段树的所有子节点之和(简单理解为某一段区间内的和)
    • 例如根节点的val所代表的含义为整个区间内的和sum
  • add为懒标记，add的取值是否为0代表了此节点下的所有孩子节点是否需要更新
    • 当add为0时，代表该区间内无需更新任何值
    • 当add不为0时，我们只需要对该节点的值进行更新；当我们向孩子节点遍历时，会对孩子节点的val进行更新；更新的值取决于区间更新或者单点更新

该方法表示向下更新孩子节点：

  • 前两行用于初始化线段树时创建节点
  • 若add为0，则不做任何更新
  • 不为0，则左节点更新mid左端元素的个数 * 需要更新的值add；右节点更新mid右端元素的个数 * 需要更新的值add
  • 将懒标记向下传递后，最后将父节点的懒标记置零

向上更新线段树：

  • 当节点更新到底层后，将更新后的区间综合val向上返回到父节点

更新线段树：

  • 该方法更新的是**[lc, rc]中的[l, r]**的结果
  • 若**[lc, rc]包含在[l, r]**中，则直接进行更新，同时更新懒标记add
  • 否则我们对区间进行二分，在左右区间内分别更新[l, r]的值；更新方法为pushUp, pushDown

线段树的查询：

  • 和更新方法类似
    • 若包含，则直接返回val
    • 否则查询mid左右的区间，加到ans中返回
