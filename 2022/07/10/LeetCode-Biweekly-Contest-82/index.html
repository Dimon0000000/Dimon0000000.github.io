Problem B：坐上公交的最晚时间

题意分析

最难Q2

给定一个公交车运行表和乘客到达时间表，返回你最晚到达且可以赶上公交的时间

贪心：

  • 如果到达最后最后一班时，前方的所有人都可以上车，且任然存在空位，我们就选择发车的时间点到达
  • 否则，我们找到最后一个上车的人，从他的到达时间逐次减一，直到找到可以的上车时间

参考题解：脑筋急转弯 (Python/Java/C++/Go)

problem C：最小差值平方和

题意分析

给定两个数组和对两个数组的修改次数k1, k2，返回一定修改次数后数组可以得到的最小平方和
注意修改次数不一定需要用完

与2233题相类似的是，这里的两个数组和两个操作次数完全可以类比成一个数组和一次性的操作

输入：nums1 = [1,1,3,4], nums2 = [1,3,3,4], k1 = 1, k2 = 1
据此我们得到的差值数组为[0, 2, 0, 0]
对于任意一个diff[i]的减少，可以从nums1[i]进行操作，也可以从nums2[i]进行，相当于可以直接在diff数组上进行 -1 操作

举个例子：

nums1[i] = 1, nums2[i] = 3, total = k1 + k2 = 2; diff[i] = 2

  • 当k1 = 1, k2 = 1; 我们可以对nums2[i]进行 -1 操作，对nums1[i]进行 +1 操作后
    • nums1[i] = 2, nums2[i] = 2, diff = 0;
  • 当k1 = 2, k2 = 0; 我们可以对nums1[i]进行两次 +1 操作
    • nums1[i] = 3, nums2[i] = 3, diff = 0;
  • 当k1 = 0, k2 = 2; 我们可以对nums2[i]进行两次 -1 操作
    • nums1[i] = 1, nums2[i] = 1, diff = 0;
  • 所有的一切反应到diff数组上，无非就是对diff[i] = 2进行了两次 -1 操作

贪心：

在暴力解法中，一次性我们只取出一个数字进行减法操作。很明显这一步是可以进行优化的；每一次可以取出最大值的所有数字 进行“-1操作”

则具体的做法呼之欲出：

  • 第一次遍历数组，维护数组中的最大值，并且将各个数字的数量存储到哈希表中
  • 从最大值开始，每一次对用总的“-1操作”次数total减去该最大值的数量，并更新哈希表中的值
  • 注意由于是平方最小值，因此我们目标的最小值为0；当最大值max达到0时，标志着平方和为0，直接结束即可
